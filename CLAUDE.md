# Paddy's Parade Day Adventure

Single-file HTML5 canvas game set during the Scranton, PA St. Patrick's Day Parade.

## Key Files
- `paddy-parade.html` ŌĆö entire game (HTML + CSS + JS, ~4450 lines) ŌĆö **generated by build.js**
- `paddy-parade.template.html` ŌĆö structural template with `%%LUCKY_COINS%%`, `%%QUESTION_BANK%%`, `%%DIALOGUES%%` markers
- `content/lucky_coins.json` ŌĆö editable Lucky Coin data (6 coins)

- `content/dialogues.json` ŌĆö editable NPC dialogue trees (6 trees)

- `paddy-parade-creator.html` ŌĆö hotspot visualizer/debugging tool
- `paddy-parade-docs.html` ŌĆö developer reference (standalone HTML)
- `validate_scenes.py` ŌĆö dev-only Python validation tool (requires Node.js)

## Build System
Content (dialogues, questions, coins) lives in `content/*.json` ŌĆö edit these for mass text changes.

**Editing content workflow:**
1. Edit `content/questions.json`, `content/dialogues.json`, or `content/lucky_coins.json`
2. Run `node build.js` (or `npm run build`) to regenerate `paddy-parade.html`
3. Open `paddy-parade.html` in browser to test
4. Commit and push ŌĆö Vercel runs `node build.js` automatically on deploy

**One-time setup (already done):**
```
node extract-content.js   # creates template + content/*.json from existing paddy-parade.html
```

**Why regex, not eval:** `const` inside `eval()` is always block-scoped and not visible outside.
`extract-content.js` uses anchored regex to extract raw JS value literals and evals each separately.

## Tech Constraints
- **No external dependencies** ŌĆö pure Canvas 2D only, no libraries, no image files
- **Single file** ŌĆö all game code lives in paddy-parade.html
- **320├Ś240 virtual canvas** scaled to viewport via CSS; `devicePixelRatio` applied at boot
- `VIEWPORT_H = 195` ŌĆö playable area; `INVENTORY_Y = 195`, `INVENTORY_H = 45` ŌĆö bottom bar

## Code Structure (10 sections)
1. Constants & Palette (`W`, `H`, `PALETTE`, `LUCKY_COINS`, `QUESTION_BANK`)
2. GameState class (flags, score)
3. Input class (pointer events ŌåÆ logical coords)
4. Sprite Helpers (`drawPaddy`, `drawNPC`, `drawShamrock`, `drawItemIcon`, `roundRect`, `wrapText`)
5. Scene Backgrounds (6 ├Ś `drawBg*` functions, all cached in `bgCache`)
6. Scene Definitions (`SCENES` object)
7. Dialogue System (`DIALOGUES` trees + `DialogueSystem` class)
8. UI System (`UISystem`: inventory, popups, HUD)
9. Game Engine (`Game` class: loop, walk, transitions, coins, help screen, difficulty)
10. Boot (`window.onload`)

## Common Gotchas
- Background functions are **cached** (drawn once to offscreen canvas). Do NOT use `Math.random()` inside them.
- `wrapText(ctx, text, x, y, maxW, lineH)` **returns the final y** ŌĆö use it to chain layout.
- `fillText` y is the **baseline**, not the top. For Npx font, top of glyphs Ōēł y ŌłÆ N.
- Quiz options must fit within canvas height 240. `by0 + 4*(optionH + optionGap) < 240`.
  - Current values: `by0=92, optionH=30, optionGap=3` ŌåÆ last option ends at y=221 Ō£ō
  - Feedback modal overlays y=142ŌĆō230 (covers last 2 options + below) ŌĆö button at y=216 Ō£ō
- The `animTimer` is incremented in `title`, `help`, `difficulty`, and `play` modes ŌĆö not in `dialogue`.
- Coins are drawn live (not in bgCache) via `_renderCoins()`.
- NPCs and Paddy are depth-sorted by Y before drawing (higher Y = closer to camera).
- `_renderAmbient(ctx)` draws animated overlays on top of bgCache (fire, steam, crowd wave, cart offset).
- `_renderExitSigns(ctx)` is called after `_renderAmbient` and before entity rendering in play mode.
- `_getHoverLabel()` returns the current sentence-line text (hovered name or action string).
- Walk target indicator uses `walkTargetTimer`, `walkTargetX/Y` in Game state.
- `drawNPC` now accepts optional 6th param `facing` (-1 or 1) ŌĆö applies ctx.scale(-1,1) for mirror.
- `quizFromBank` nodes in DIALOGUES pull from `game.sessionQuestions[scene]` by pointer index.
- Score multiplier: Easy=1, Medium=1.5, Hard=2 (applied in `_handleQuizTap`).
- Quiz feedback uses tap-to-dismiss (no timer) ŌĆö `DialogueSystem.update(dt)` is a no-op.
