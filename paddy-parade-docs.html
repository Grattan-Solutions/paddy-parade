☘ Paddy's Parade Day

Developer Reference — Scranton St. Patrick's Parade Adventure
Pure Canvas 2D No dependencies Single HTML file Mobile-friendly 320×240 virtual resolution

1. Overview

Paddy's Parade Day Adventure is a point-and-click adventure game built entirely in a single HTML file using the HTML5 Canvas 2D API. No external libraries, no build tools, no image assets — all graphics are drawn procedurally at runtime.

Tech Stack

  • Rendering: HTML5 Canvas 2D (320×240 virtual pixels, scaled to viewport)
  • Input: Unified Pointer Events API (works on mouse, touch, and stylus)
  • Animation: requestAnimationFrame game loop
  • State: Plain JavaScript classes (no framework)
  • Async flow: async/await with manual Promise resolution for dialogue

Intended Audience

This doc is for developers who want to extend the game — adding scenes, NPCs, quiz questions, or lucky coins. It assumes basic JavaScript knowledge but does not require any game engine experience.

2. How to Play (Player-Facing)

The Story

It's St. Patrick's Day morning in Scranton, PA. Bartender Siobhan O'Malley tells Paddy the Leprechaun that the St. Patrick's Parade is in danger — four essential items are missing! Paddy must explore five Scranton landmarks, answer history quiz questions to earn the items, and deliver them all to Grand Marshal O'Brien before noon.

Controls

ActionWhat happens
Tap a person (NPC)Paddy walks over and initiates dialogue
Tap a highlighted objectVerb popup appears (Look / Take / Go / etc.)
Tap empty floorPaddy walks to that location
Tap an inventory iconSelects the item; then tap a hotspot to USE it
Tap a gold coinCollects the Lucky Coin (+5 pts, shows fun fact)

Scoring

  • +10 points per correct quiz answer (10 quiz questions total = 80 max quiz points)
  • +5 points per Lucky Coin collected (5 coins total = 25 max coin points)
  • Maximum score: 105 points
  • Wrong quiz answers can be retried (no point deduction)

4. Scene System

Each scene is a plain JavaScript object in the SCENES map (Section 6). Scenes connect to each other via exit hotspots.

Scene Object Shape

// Example: pub scene
const SCENES = {
  pub: {
    id:         'pub',            // Must match key
    name:       "Paddy's Pub",   // Shown in scene banner
    paddyStart: { x: 80, y: 155 }, // Paddy's entry position
    drawBg:     drawBgPub,          // Background render function
    npcs:       [{ id: 'siobhan', type: 'siobhan', x: 240, y: 155, dialogueId: 'siobhan' }],
    hotspots:   [ ... ],
    dialogues:  { ... },
  },
};

5. Dialogue System

Dialogue trees live in the DIALOGUES object (Section 7). Each top-level key is a dialogueId; its value is a map of node IDs to node objects.

Node Types

PropertyTypeBehaviour
linesstring[]Displays each line in the dialogue box; player taps to advance
quizobjectShows a multiple-choice quiz question
nextstringNode ID to run after this node completes
doneFlagstringState flag set to true when this node runs
checkItemsboolChecks if all 4 items are in inventory; branches to items_ready or items_missing
winGameboolSets parade_started flag, triggers win screen

Quiz Node Shape

quiz1: {
  quiz: {
    question: "What color is most associated with St. Patrick's Day?",
    options:  ['Green', 'Blue', 'Red', 'Orange'],
    correct:  0,            // Index into options (0-based)
    hint:     "Think of shamrocks!",
    flag:     'quiz_pub_q1',  // Set to true on correct answer
  },
  next: 'quiz2',
},

Async Flow

DialogueSystem.run() is an async method. It calls await this.runNode(tree, nodeId) recursively. Each showLines() and showQuiz() returns a Promise that resolves only when the player acts (taps through lines, or answers a quiz correctly).

The game mode is set to 'dialogue' during this time. Rendering and tap handling in update() are delegated to DialogueSystem.update() and DialogueSystem.handleTap().

7. Lucky Coins

Lucky Coins are hidden gold coins placed in each scene. They're subtle — slightly visible but easy to miss — and reward exploration with +5 points and an Irish history fun fact.

Data Structure

const LUCKY_COINS = [
  {
    id:       'coin_pub',    // Unique ID (also the state flag name)
    scene:    'pub',         // Must match a SCENES key
    x:        58,              // Canvas X position
    y:        125,             // Canvas Y position (must be < VIEWPORT_H = 195)
    dialogue: "☘ Fun Fact: ...", // Text shown in dialogue box on collect
  },
  // ... one per scene
];

Coin Positions

Rendering

Coins are drawn in Game._renderCoins(ctx) every frame (not in the background cache). This lets them disappear when collected without invalidating the background. The glint sparkle animation uses Math.floor(animTimer * 60) % 90 so it fires once every 1.5 seconds.

Hit Detection

In _handlePlayTap(x, y), coins are checked before hotspots (step 4b). A coin is hit when dx² + dy² < 100 (radius = 10px). Collected coins are flagged in state and skipped in both rendering and hit detection.

Adding a New Lucky Coin

  1. Pick a scene ID from SCENES keys
  2. Pick x/y coordinates (must be within 0–320 and 0–195)
  3. Write a fun Irish-American history fact
  4. Add an entry to the LUCKY_COINS array at the top of Section 1
  5. That's it — detection, rendering, and collection are automatic

8. Adding New Content

Add a New Scene

  1. Write a drawBgMyScene(ctx) function in Section 5. Use only Canvas 2D — no image loading.
  2. Add an entry to SCENES in Section 6: 
    myScene: {
  id: 'myScene',
  name: "My New Location",
  paddyStart: { x: 80, y: 155 },
  drawBg: drawBgMyScene,
  npcs: [],       // or add an NPC
  hotspots: [
    { id: 'exit_back', name: 'Back to Pub', rect: makeRect(0,100,20,95),
      verbs: ['use'], isExit: true, exitTo: 'pub' },
  ],
  dialogues: {},
},
  3. Add an exit hotspot in an existing scene pointing to exitTo: 'myScene'.
  4. Optionally add a Lucky Coin for the scene.

Add a New NPC

  1. Add a new else if (type === 'myNpc') block inside drawNPC() in Section 4. The character is drawn centered at (0, 0) where y=0 is the floor/foot level.
  2. Create a dialogue tree in DIALOGUES: 
    myNpc: {
  root: {
    lines: ["Hello, Paddy!"],
    next: 'quiz1',
  },
  quiz1: { quiz: { question: "...", options: [...], correct: 0, hint: "...", flag: 'quiz_my_q1' }, next: 'farewell' },
  farewell: { lines: ["Good luck!"], doneFlag: 'quiz_my_done' },
  already_done: { lines: ["See you later!"] },
},
  3. Add the NPC to the scene's npcs array: { id: 'myNpc', type: 'myNpc', x: 200, y: 155, dialogueId: 'myNpc' }
  4. Add the NPC's display name to Game._npcDisplayName()

Add a New Quiz Question

  1. Choose which NPC's dialogue to extend
  2. Add a new quiz node (e.g. quiz3) after quiz2
  3. Change quiz2.next from 'farewell' to 'quiz3'
  4. Set quiz3.next: 'farewell'
  5. Each correct answer is worth +10 pts automatically

Add a New Lucky Coin

// In Section 1, add to LUCKY_COINS array:
{ id: 'coin_myScene', scene: 'myScene', x: 150, y: 120,
  dialogue: "☘ Fun Fact: ..." }

That's all — no other code changes needed.

Coordinate System

  • Canvas: 320×240 px virtual, scaled to viewport. Origin top-left.
  • Viewport height: VIEWPORT_H = 195 — the playfield. Below this is the inventory bar.
  • Inventory bar: INVENTORY_Y = 195, height = 45px.
  • Sprite origin: Characters are drawn at (x, y) where y = ground/foot level. All offsets are negative (upward).

Background Caching

Each scene's background is rendered once into an offscreen canvas (bgCache) and then blitted each frame with ctx.drawImage(bgCache, 0, 0). The cache is invalidated (bgCache = null) on scene transitions. Lucky Coins and NPCs are drawn after the cached image, so they are never included in the cache.

Tip: Avoid using Math.random() in background drawing functions — since the background is cached on the first frame, you'd get a different random result each time the cache is invalidated. Use a seeded pattern instead (e.g., deterministic loops).
Mobile: The canvas scales to fit the viewport using min(100vw, 100vh * 4/3). Tap events work the same as mouse events. The Input class normalizes coordinates back to virtual 320×240 space.
